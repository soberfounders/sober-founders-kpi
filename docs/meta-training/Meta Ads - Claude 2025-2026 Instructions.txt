1.2 Meta Pixel + Conversions API (CAPI) Setup
PRIORITY INSTRUCTION: Both Pixel AND CAPI must be implemented together. This is non-negotiable in 2025.
Meta Pixel (Client-Side)
PURPOSE: Browser-based event tracking
CAPTURES: Page views, add-to-cart, real-time behavior, user agent, referral source
LIMITATIONS: Blocked by ad blockers, iOS privacy settings, Safari ITP, cookie restrictions
RESULT: 40% or lower attribution accuracy when used alone
Conversions API (Server-Side)
PURPOSE: Server-to-server event tracking
CAPTURES: Purchases, leads, subscriptions, offline conversions
ADVANTAGES: Works regardless of browser settings, cookie policies, privacy restrictions
IMPROVES: Event Match Quality (EMQ), attribution accuracy, iOS tracking
Implementation Steps:
Install Meta Pixel code in website <head> section
Configure standard events: PageView, ViewContent, AddToCart, InitiateCheckout, Purchase
Generate CAPI Access Token in Events Manager > Conversions API > Settings
Implement server-side CAPI integration (use platform integrations: Shopify, WordPress, Segment)
CRITICAL: Set unique event_id for each event in both Pixel and CAPI
Meta uses event_id + timestamp for deduplication (prevents double-counting)
Send advanced matching parameters: email (hashed), phone (hashed), first name, last name, city, state, zip, country
Monitor Event Match Quality score in Events Manager (aim for 8.0+)
Key Tracking Metrics:
Event Match Quality (EMQ): 8.0+ is excellent
Events Received: Both Pixel and CAPI should show events
Deduplication Status: Check Diagnostics tab
Signal Quality: Higher = better optimization
Common Setup Errors:
Missing deduplication keys (causes double-counting or lost events)
Incomplete user identifiers (hurts EMQ and attribution)
Payment on third-party domain (Pixel loses context; CAPI required)
Over-sending low-value events (clutters reports)
Under-sending critical conversion events
1.3 Attribution Settings

SECTION 2: CAMPAIGN STRUCTURE (2025 FRAMEWORK)
2.1 The Consolidation Principle
CRITICAL RULE: Fewer campaigns = Better performance
Why Consolidation Works:
Meta algorithm needs 50+ conversions per ad set per week to optimize
Splitting audiences divides conversion data
Slows learning phase
Increases costs
Creates audience overlap and auction competition against yourself
The 2025 Campaign Structure:
RECOMMENDED STRUCTURE:
├── Campaign 1: Prospecting (Sales Objective)
│   └── Ad Set 1: Consolidated Cold Audience
│       ├── Ad 1: UGC Video Variant A
│       ├── Ad 2: UGC Video Variant B
│       ├── Ad 3: Static Image Variant
│       ├── Ad 4: Carousel Variant
│       └── Ad 5-20: Additional creative variations
│
├── Campaign 2: Retargeting (Sales Objective)
│   └── Ad Set 1: Consolidated Warm Audience
│       ├── Ad 1: Social Proof Video
│       ├── Ad 2: Testimonial
│       ├── Ad 3: Offer/Promotion
│       └── Ad 4-10: Additional variations
│
└── Campaign 3: Testing (Optional - Sales Objective)
└── Ad Set 1: New Creative Concepts
└── Ads: Experimental formats
Consolidation Best Practices:
Combine multiple interest audiences into single ad set
Layer lookalike audiences together (don't separate)
Use broad targeting or Advantage+ Audience
Exclude warm audiences from prospecting campaigns (prevent overlap)
One campaign per primary objective
Multiple creatives within ad sets (3-20 depending on budget)

SECTION 5: BUDGET & BIDDING STRATEGY
5.1 Budget Fundamentals
Minimum Budget Guidelines:
Test phase: $20-50/day per ad set
Scale phase: $50-200+/day
General rule: 10x your target CPA as daily budget minimum
Example: $25 target CPA = $250/day minimum for optimal learning
Budget Allocation (55/25/15/5 Framework):
55%: Proven campaigns (known winners)
25%: Audience testing (new segments)
15%: Creative testing (new formats/concepts)
5%: Platform testing (experimental placements)
Daily vs Lifetime Budget:
Daily Budget: Consistent spend each day, predictable
Lifetime Budget: Meta distributes across date range, can frontload
Recommendation: Daily budget for most campaigns (easier control)
5.2 Bidding Strategies
Primary Bidding Options:
Highest Volume (Lowest Cost) - DEFAULT
Use When: Starting new campaigns, flexible on CPA, want maximum conversions
How It Works: Meta bids to get maximum conversions within budget
Best For: Campaigns with flexible CPA, scale-focused, gathering data
Recommendation: Start here for first 7-14 days
Cost Cap
Use When: Want to control average CPA while scaling
How It Works: Set target average CPA, Meta adjusts bids to maintain average (individual conversions may vary)
Best For: Scaling campaigns, balance between cost and volume
Implementation: Set cost cap 10-20% below current average CPA, monitor spend
Note: If not spending, raise cost cap incrementally or improve creative
Bid Cap
Use When: Need strict control over maximum bid per auction
How It Works: Hard limit on bid amount, won't exceed in any auction
Best For: Competitive markets, preventing overspend, strict budget control
Tradeoff: May significantly reduce delivery volume
Caution: Can prevent spending if set too low, requires frequent adjustment
ROAS Goal (Minimum ROAS)
Use When: Optimizing for Value, need minimum return on ad spend
How It Works: Set minimum ROAS target (e.g., 3.0 = $3 revenue per $1 spend), Meta only bids on opportunities meeting threshold
Best For: E-commerce, clear revenue tracking, profitability focus
Requirements: Value optimization, accurate purchase value tracking, higher budgets work best
Implementation: Start with realistic ROAS based on historical data, adjust gradually
Target Cost (Legacy/Phasing Out)
Use When: Want consistent CPA, predictability over time
How It Works: Maintains target CPA as consistently as possible
Note: Being phased out in favor of Cost Cap
When Still Useful: Tight budget constraints, need predictable costs
5.3 Bidding Strategy Selection Framework
IF: Just starting, learning about audience
THEN: Highest Volume (7-14 days minimum)
IF: Have conversion data, want to scale efficiently
THEN: Cost Cap (set at 10-20% below current avg CPA)
IF: Profit margins tight, strict CPA requirements
THEN: Bid Cap (set based on max acceptable CPA)
IF: E-commerce with revenue tracking, profitability focus
THEN: ROAS Goal (start conservative, 20-30% below current ROAS)
IF: Testing new creative/audience
THEN: Highest Volume (gather data first)
IF: Black Friday/competitive periods
THEN: Bid Cap (control costs during volatility)
5.4 Budget Scaling Strategy
The 20% Rule (Safe Scaling):
Increase budget by maximum 20-25% every 3-4 days
Allows algorithm to re-learn without major disruption
Monitor performance for 48-72 hours after each increase
If performance degrades, pause increases or scale back
Scaling Indicators (When to Scale):
ROAS consistently meeting/exceeding targets for 14+ days
50+ optimization events per week
Stable or decreasing CPA over time
Creative not showing fatigue (CTR stable)
Ad set exited learning phase
Scaling Methods:
Vertical Scaling (Increase Budget)
Increase budget on winning ad sets
Follow 20% rule
Monitor learning phase resets
Horizontal Scaling (Duplicate)
Duplicate winning campaigns/ad sets
Change ONE variable (audience OR creative)
Maintain separate budgets
Monitor for audience overlap
Campaign Budget Optimization (CBO)
Set budget at campaign level
Meta distributes across ad sets automatically
Good for multiple ad sets within campaign
Less control over individual ad set spend
Scaling Pitfalls to Avoid:
Increasing budget >25% at once (triggers re-learning)
Scaling too quickly when reaching saturation
Not monitoring frequency (>3 = fatigue setting in)
Ignoring creative refresh during scaling
SECTION 6: OPTIMIZATION & MONITORING
6.1 Learning Phase
What It Is:
Period when Meta's algorithm gathers data about ad performance
Typically requires 50 optimization events (conversions) within 7 days
Performance can fluctuate during this phase
How to Exit Learning Faster:
Consolidate campaigns (more budget per ad set)
Avoid mid-campaign edits
Use proven creative from start
Ensure proper tracking (Pixel + CAPI)
Give campaigns 5-7 days minimum before major changes
What Resets Learning Phase:
Budget changes >20%
Targeting changes
Creative changes (adding/removing ads)
Optimization event changes
Bid strategy changes
Best Practice: Make changes sparingly during learning
6.2 Key Performance Metrics
Primary Metrics (Track Daily):
ROAS (Return on Ad Spend)
Formula: Revenue / Ad Spend
Target: Varies by business (typically 2-5x)
Most important profitability metric
CPA (Cost Per Acquisition)
Formula: Ad Spend / Conversions
Compare to target/breakeven CPA
Lower = better (if quality maintained)
CTR (Click-Through Rate)
Formula: Clicks / Impressions
Benchmark: 1-3% is good, >3% is excellent
Indicates ad relevance and creative quality
CPM (Cost Per Mille/1000 Impressions)
Shows auction competitiveness
Rising CPM = increasing competition or ad fatigue
Benchmark: $10-30 typical, varies by industry
Frequency
Average times each person saw ad
Target: <3 for cold audiences, <5 for retargeting
3 = fatigue risk, refresh creative
Hook Rate (3-Second Video Views)
Percentage who watch first 3 seconds
Measures hook effectiveness
Higher = better creative hook
Secondary Metrics:
Event Match Quality (EMQ): Track in Events Manager, aim for 8.0+
Conversion Rate: Landing page effectiveness
AOV (Average Order Value): Revenue per purchase
Relevance Score: Meta's quality rating (aim for Above Average)
Outbound CTR: Clicks to website / impressions
6.3 Daily Optimization Checklist
Every 24-48 Hours:
Check budget spend (on track?)
Review ROAS and CPA (meeting targets?)
Check frequency (>3? refresh creative)
Monitor CTR (declining? test new creative)
Review CPM trends (increasing? investigate why)
Check learning phase status
Review Event Match Quality in Events Manager
Monitor for any account/pixel errors
Weekly:
Analyze performance by ad, ad set, campaign
Pause underperformers (CPA >50% above target for 7+ days)
Scale winners (increase budget 20% if ROAS strong)
Test new creative variations
Review audience overlap
Check placement performance breakdown
Analyze by demographics, devices, locations
Review and update creative queue
Monthly:
Full account audit
Analyze trends across all campaigns
Budget reallocation based on performance
Major creative refresh/strategy shifts
Competitor research
Update lookalike audiences with fresh data
Review and optimize landing pages
Test new campaign structures
6.4 When to Pause vs Optimize
Pause When:
CPA >100% above target for 7+ consecutive days
ROAS <50% of target for 7+ days
Spent >3x target CPA without conversion
Frequency >5 with declining performance
Ad set stuck in learning with <50 conversions in 2 weeks
Optimize Before Pausing:
Test new creative (most common issue)
Adjust bid strategy
Expand audience (if too narrow)
Remove worst-performing placements
Improve landing page experience
Check tracking (Pixel/CAPI working?)
Never Blame Meta First:
90% of issues are within your control
Most problems: creative fatigue, poor targeting, landing page issues, tracking problems
Take responsibility, adjust, retest
SECTION 7: ADVANCED STRATEGIES
7.1 Full-Funnel Strategy
Top of Funnel (ToFu) - Awareness
Goal: Introduce brand to cold audiences
Creative: Educational content, pain point focus, story-driven, value-first
Messaging: "Why this problem matters" not "buy now"
Audience: Broad, interest-based, cold
Metrics: Engagement, video views, reach, brand lift
Middle of Funnel (MoFu) - Consideration
Goal: Nurture interest, build trust
Creative: Product benefits, how-it-works, comparisons, social proof
Messaging: "Why our solution" and "what makes us different"
Audience: Website visitors, video viewers, engagers
Metrics: CTR, landing page views, add-to-cart
Bottom of Funnel (BoFu) - Conversion
Goal: Drive purchase decision
Creative: Testimonials, reviews, case studies, offers, urgency
Messaging: "Buy now" with trust builders and incentives
Audience: Cart abandoners, product viewers, high-intent signals
Metrics: ROAS, CPA, purchases, conversion rate
2025 Update: Consolidated Funnels
New Best Practice: Combine cold, warm, hot in ONE campaign
Why: Faster learning, better optimization, higher returns
How: Use different ad creative for different stages within same campaign
Exception: Still separate if distinctly different offers or products

7.3 Dynamic Creative Optimization (DCO)
What It Is:
Meta automatically tests combinations of creative elements
Mixes headlines, images, videos, descriptions, CTAs
Delivers best-performing combinations to each user
How to Use:
Create multiple asset variations:
3-5 headlines
3-5 primary text variations
3-5 images or videos
2-3 CTAs
Meta tests all combinations
Algorithm learns and optimizes delivery
View results by asset in Ads Manager
Best Practices:
Provide high-quality, diverse assets
Don't mix drastically different concepts (confuses algorithm)
Give DCO 7-14 days to learn
Review asset performance breakdown
Remove worst performers, add new variations

Optimize for Value (Not Conversions):
In ad set, optimize for "Value" not "Conversions"
Meta targets high-value purchasers
May get fewer conversions but higher revenue
Set Minimum ROAS Goal:
Optional: Set minimum ROAS in bid strategy
Example: 3.0 ROAS minimum
Meta only bids on opportunities likely to meet threshold
Use Case:
Products with wide price ranges
High vs low lifetime value customers
Focus on revenue not just conversion count
7.5 Seasonal/Event-Based Strategy
Preparation (4-6 Weeks Before):
Build custom audiences of engaged users
Test creative concepts early
Gather social proof and UGC
Optimize landing pages
Ensure tracking is flawless
During Peak (Black Friday, Holidays):
Use Bid Cap strategy (control costs during high CPMs)
Increase budgets gradually leading up to event
Launch 7-10 days before peak day
Focus on proven winners (not time for major tests)
Monitor hourly during peak times
Adjust bids based on auction competitiveness
Post-Event:
Don't immediately cut budgets to zero

SECTION 8: COMMON MISTAKES & HOW TO AVOID
8.1 Critical Errors (Fix Immediately)
Pixel/CAPI Not Set Up Properly
Result: Blind optimization, wasted spend
Fix: Verify Events Manager shows events, check EMQ score, test purchase flow
Prevention: Test tracking before spending significant budget
Over-Segmenting Campaigns
Error: Multiple ad sets with same objective, fractured conversion data
Result: Slow learning, higher costs, audience overlap
Fix: Consolidate into 1-2 ad sets per campaign
Prevention: Follow consolidation principle
Ignoring Creative Quality
Error: Using stock photos, overly branded content, boring hooks
Result: Poor performance regardless of budget or targeting
Fix: Invest in UGC-style content, test hooks, refresh regularly
Prevention: Allocate 40%+ of budget to creative production/testing
Making Too Many Changes During Learning
Error: Adjusting budgets, targeting, creative every day
Result: Constant learning phase resets, never optimizing
Fix: Let campaigns run 5-7 days before major changes
Prevention: Plan campaigns carefully, be patient
Not Excluding Existing Customers
Error: Prospecting campaigns target people who already bought
Result: Wasted spend, overlap with retention efforts
Fix: Create customer list exclusion, apply to prospecting
Prevention: Always check exclusions before launch
Scaling Too Fast
Error: Doubling/tripling budgets overnight
Result: Learning reset, performance crash, wasted spend
Fix: Scale back, increase 20% every 3-4 days
Prevention: Follow 20% scaling rule religiously
Ignoring Frequency
Error: Same people see same ad 10+ times
Result: Ad fatigue, declining CTR, rising costs
Fix: Refresh creative, expand audience, cap frequency
Prevention: Monitor frequency daily, refresh at >3
Poor Landing Page Experience
Error: Slow load times, confusing navigation, weak copy
Result: Great CTR but terrible conversion rate, wasted clicks
Fix: Optimize page speed, clarify offer, simplify checkout
Prevention: Test landing page before launching ads
8.2 Common Beginner Mistakes
Expecting Immediate Results
Reality: Learning phase takes 5-7 days minimum
Solution: Give campaigns time, plan for 2-week initial testing
Copying Competitors Exactly
Reality: Their strategy may not fit your business
Solution: Learn principles, adapt to your unique value prop
Blaming the Algorithm
Reality: 90% of issues are within your control
Solution: Check creative, tracking, landing page before blaming Meta
Not Testing Enough Creative
Reality: Creative = 70-80% of performance
Solution: Test 3-5 new creatives per month minimum
Setting Unrealistic Targets
Reality: Market conditions, competition affect results
Solution: Research industry benchmarks, start conservative
Ignoring Attribution Windows
Reality: Different windows = different reported results
Solution: Pick one setting, stick with it, understand its limitations
Not Using Advantage+ Features
Reality: Meta's AI is now better than manual in most cases
Solution: Test Advantage+ against manual, let data decide
SECTION 9: TESTING FRAMEWORK
9.1 Sequential Testing Methodology
Test ONE Variable at a Time:
Changing multiple variables = can't identify what worked
Run tests sequentially, not simultaneously
Testing Priority Order:
Creative First (Weeks 1-4)
Hook variations (most impactful)
Format variations (video vs static vs carousel)

9.2 Statistical Significance
Minimum Test Requirements:
50+ conversions per variation minimum
7+ days of data
Stable performance (no major fluctuations)
Decision Framework:
IF: Variation A has >20% better CPA over 7 days with 50+ conversions
THEN: Declare winner, pause loser
IF: Difference <20% after 14 days
THEN: No clear winner, keep both or retest with bigger difference
IF: One variation has 2x better ROAS
THEN: Instant winner regardless of timeline

SECTION 10: TROUBLESHOOTING GUIDE
10.1 Low Spend Issues
Symptom: Campaign spending <50% of daily budget
Causes & Fixes:
Audience Too Narrow
Check: Estimated audience size <100k?
Fix: Broaden targeting, add interests, use Advantage+ Audience
Bid Too Low
Check: Using Bid Cap below market rate?
Fix: Increase bid cap by 20%, switch to Highest Volume temporarily
Ad Quality/Relevance Low
Check: Low CTR, high negative feedback?
Fix: Refresh creative, improve hook, test new angles
Budget Too High for Audience
Check: Budget >10x daily conversion potential?
Fix: Reduce budget or expand audience
Account in Learning
Check: Recent account/campaign creation?
Fix: Give 5-7 days, ensure 50+ conversions weekly
10.2 High Cost Issues
Symptom: CPA 2-3x higher than expected
Causes & Fixes:
Creative Fatigue
Check: Frequency >3, declining CTR?
Fix: Refresh creative immediately, rotate new ads
Wrong Audience
Check: Targeting too broad or too niche?
Fix: Use custom audiences, refine targeting
Poor Landing Page
Check: High CTR but low conversion rate?
Fix: Optimize landing page, speed, clarity, offer
Competitive Market
Check: CPMs 50-100% higher than industry average?
Fix: Improve creative quality, test niche placements, consider timing
Tracking Issues
Check: Low Event Match Quality, missing events?
Fix: Fix Pixel/CAPI implementation, verify attribution
Wrong Optimization Event
Check: Optimizing for Add to Cart instead of Purchase?
Fix: Optimize for event closest to actual goal
10.3 Low Conversion Rate
Symptom: Good traffic, poor conversions
Causes & Fixes:
Landing Page Mismatch
Check: Ad promise ≠ landing page content?
Fix: Align messaging, maintain scent from ad to page
Poor Page Experience
Check: Slow load time (>3 seconds), confusing layout?
Fix: Optimize speed, simplify design, clear CTA
Wrong Audience
Check: High bounce rate, low time on page?
Fix: Refine targeting, ensure ad attracts right people
Weak Offer
Check: Competitors have better pricing/value?
Fix: Strengthen offer, add bonuses, guarantees, urgency
Trust Signals Missing
Check: No reviews, testimonials, security badges?

Check browser console for JavaScript errors

SECTION 13: CONFLICTING INFORMATION (OPINIONS VS FACTS)
Areas of Disagreement/Debate
1. Campaign Budget Optimization (CBO) vs Ad Set Budget Optimization (ABO)
Position A: CBO is Superior
Arguments: Meta AI distributes budget optimally, less manual work, scales better
Proponents: Advocates for full automation, trust algorithm
Position B: ABO Gives Better Control
Arguments: Control exactly where spend goes, prevents overspending on poor ad sets, more transparency
Proponents: Manual optimization experts, those burned by CBO misallocation
Likely Reality: Context-dependent
CBO works better when you trust Meta's AI and have proven ad sets
ABO better for testing, niche audiences, strict budget controls
Hybrid approach (use both for different campaigns) common among advanced advertisers
2. Advantage+ Sales Campaigns (ASC) vs Manual Campaigns
Position A: ASC is the Future
Arguments: Latest AI features, faster learning, better results for most advertisers, Meta prioritizes with better delivery
Proponents: Automation advocates, data showing ASC outperformance
Position B: Manual Campaigns Still Superior for Control
Arguments: Transparency, precise targeting, no black-box decisions, better for niche markets
Proponents: Experienced advertisers who've optimized manual campaigns for years
Likely Reality: Both have use cases
ASC excellent for scale and speed with standard products
Manual better for niche targeting, complex funnels, brand safety
Most successful advertisers run both
3. Broad Targeting vs Interest-Based Targeting
Position A: Always Use Broad (Advantage+ Audience)
Arguments: AI is smarter than manual targeting, interest targeting outdated, creative teaches algorithm
Proponents: 2025 best practice advocates, Meta official guidance
Position B: Interest Targeting Still Works
Arguments: Provides guardrails, prevents wasted spend, better for niche products, faster initial results
Proponents: Conservative optimizers, niche market advertisers
Likely Reality: Broad is winning for most, but not all
Broad + strong creative = best practice for mainstream products
Interest targeting still useful for hyper-niche products, regional businesses, extremely specific audiences
Starting broad and narrowing if needed > starting narrow
4. Optimal Number of Ads Per Ad Set
Position A: 3-5 Ads Maximum
Arguments: Clear performance differentiation, easier to identify winners, faster learning
Proponents: Simplicity advocates, conservative testers
Position B: 10-20+ Ads for Volume
Arguments: More variety = more signals, prevents fatigue, reaches different segments
Proponents: High-budget advertisers, creative diversity advocates
Likely Reality: Budget-dependent
Lower budgets (<$50/day): 3-5 ads sufficient
Higher budgets ($100+/day): 10-20+ ads beneficial
Quality > quantity always applies
5. Retargeting Campaign Separation
Position A: Always Separate Prospecting and Retargeting
Arguments: Different messaging, different budgets, clear attribution, prevents overlap
Proponents: Traditional funnel marketers, clear reporting advocates
Position B: Consolidate All Audiences in One Campaign
Arguments: Faster learning, better optimization, algorithm handles segmentation, reduces complexity
Proponents: 2025 consolidation advocates, trust algorithm fully
Likely Reality: Hybrid emerging
Still separate for drastically different offers/messaging
Consolidating cold + warm works well with creative diversity
Test both approaches for your business
6. Video Length (15s vs 30s vs 60s+)
Position A: Keep Videos Short (15-30s)
Arguments: Attention spans short, mobile scrolling fast, retention drops after 15s
Proponents: Mobile-first marketers, data showing drop-off
Position B: Longer Videos (60s+) Build Better Audiences
Arguments: Quality engagement signals, storytelling works, committed viewers convert better
Proponents: Brand builders, complex product sellers
Likely Reality: Product and funnel dependent
Impulse/low-consideration products: 15-30s
Complex/high-ticket products: 60s+ for education
Test both, let thumb-stop rate and conversion data decide
7. Budget Minimum for Meta Ads
Position A: $1,000+/month Minimum
Arguments: Algorithm needs volume, <$1k wastes time, won't exit learning phase
Proponents: Agency owners, high-budget advertisers
Position B: Can Start with $10-20/day
Arguments: Small businesses need to test, gradual scaling possible, better than no advertising
Proponents: Small business advocates, budget-conscious marketers
Likely Reality: Both right with caveats
$1,000+/month ideal for optimal results and fast learning
$10-20/day possible but requires patience, excellent creative, realistic expectations
Sweet spot: $50-100/day for meaningful testing
8. When to Use Bid Caps vs Cost Caps
Position A: Cost Cap is Superior
Arguments: More flexible, still scales, better balance of cost and volume
Proponents: Most 2025 guides, Meta official recommendation
Position B: Bid Cap for Better Control
Arguments: Prevents overspending, essential in competitive markets, protects margins
Proponents: Conservative advertisers, tight-margin businesses
Likely Reality: Use case specific
Cost Cap: Default for most, scales better
Bid Cap: Competitive periods, strict CPA requirements, low-margin products
Start with Highest Volume, graduate to Cost Cap, use Bid Cap only when needed
9. Placement Strategy
Position A: Always Use Advantage+ Placements
Arguments: AI optimizes better, data proves lower costs, manual selection limits reach
Proponents: Automation-first marketers, Meta official guidance
Position B: Exclude Low-Performing Placements
Arguments: Audience Network often poor quality, brand safety concerns, specific formats work better
Proponents: Quality-focused advertisers, brand safety advocates
Likely Reality: Test and decide based on data
Start with Advantage+ Placements
Review breakdown after 30 days
Exclude only if data shows consistent poor performance
Audience Network most commonly excluded
10. Attribution Window
Position A: Use Longest Window (7-day click, 1-day view)
Arguments: Captures full customer journey, more accurate attribution, justifies spend
Proponents: Long sales cycle businesses, full-funnel marketers
Position B: Use Shorter Window (1-day click only)
Arguments: More conservative, prevents over-attribution, focuses on direct response
Proponents: Direct response marketers, conservative budgeters
Likely Reality: Align with actual customer journey
Long consideration (B2B, high-ticket): 7-day click
Short consideration (impulse, low-ticket): 1-day click
Consistency matters more than specific window
Understand limitations of any attribution model
